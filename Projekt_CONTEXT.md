# Projekt_CONTEXT.md — Архитектура и Контекст Проекта

Этот документ представляет собой исчерпывающий технический обзор архитектуры, принятых решений и текущего состояния разработчика портфолио-сайта `almanex.github.io`.

---

## 1. Суть проекта и Стек

Приложение представляет собой мультиязычный интерактивный сайт-портфолио разработчика с ретро-терминальным дизайном. Сайт содержит разделы с информацией о разработчике, списком проектов с рендерингом README, блогом и формой обратной связи, стилизованной под терминал.

### Стек технологий:
*   **Фреймворк:** [Astro v7.0.7](https://astro.build) (в режиме генерации статических сайтов — SSG)
*   **Среда выполнения:** Node.js (рекомендуется `>=22.12.0`)
*   **Стилизация:** Чистый CSS (Vanilla CSS) без внешних UI-фреймворков и библиотек стилей. Глобальный дизайн-система описан в одном файле.
*   **Парсинг Markdown:** [Marked v18.0.6](https://github.com/markedjs/marked) (используется для динамического рендеринга встроенных файлов описания проектов)
*   **Интеграции:** `@astrojs/sitemap v3.7.3` (автоматическая генерация карты сайта)
*   **Обработка форм:** Интеграция с внешним сервисом [Formspree](https://formspree.io) для сбора обратной связи.

## 2. Архитектура и Паттерны

### Структура проекта:
```text
├── .agents/               # Настройки ИИ-агентов и правила разработки
├── src/
│   ├── components/        # Презентационные компоненты страниц (Index, About, Blog и т.д.)
│   ├── content/           # Контент блогов, разделенный по языкам (en, ru, de)
│   │   ├── blog/
│   │   │   ├── de/
│   │   │   ├── ru/
│   │   │   └── wsl-containers.md
│   │   └── config.ts      # Схемы коллекций контента (Astro Content Collections)
│   ├── data/              # Статические данные (проекты, переводы интерфейса)
│   │   ├── projects.js    # База данных проектов со встроенными README
│   │   └── translations.js# Словари для мультиязычности (i18n)
│   ├── layouts/           # Глобальный шаблон разметки Layout.astro
│   ├── pages/             # Точки входа маршрутизации Astro
│   │   ├── ru/            # Локализованные русские страницы
│   │   ├── de/            # Локализованные немецкие страницы
│   │   └── ...            # Английские страницы (язык по умолчанию)
│   └── styles/            # Стили проекта (global.css)
```

### Архитектурные паттерны:
1.  **Локализация по префиксам путей (Astro i18n):**
    Astro настроен на использование трех локалей (`en`, `ru`, `de`). Маршруты в папках `ru/` и `de/` служат тонкими обертками. Они импортируют общие презентационные компоненты из `src/components/` и передают им параметр `lang="ru"` или `lang="de"`. Путь без префикса рендерится с `lang="en"`.
2.  **Шаблон Layout-Slot:**
    Вся разметка структуры (Header, Footer, мета-теги, SEO, JSON-LD) инкапсулирована в `src/layouts/Layout.astro`. Специфичный контент страниц рендерится через тег `<slot />`.
3.  **Монолитный стек данных проектов:**
    Для исключения лишних сетевых запросов и лимитов GitHub API, база проектов хранится локально в `src/data/projects.js`. Это объект, в который напрямую внедрены тексты README в виде шаблонных строк на трех языках.
4.  **Управление состоянием тем оформления:**
    Тема (светлая/темная) переключается через установку атрибута `data-theme` на уровне `document.documentElement`. Для предотвращения эффекта мерцания (Flash of Unstyled Content) в `<head>` встроен синхронный блокирующий скрипт чтения состояния из `localStorage`.

## 3. Реализованная логика (Current State)

### Маршрутизация и Языки:
*   Реализована поддержка динамических роутов для блогов:
    *   `/blog/[slug]` и локализованные аналоги (например, `/ru/blog/[slug]`).
    *   Пагинация блогов по страницам: `/blog/[...page]`.
*   Генерация SEO-тегов и разметки:
    *   Канонические ссылки (`canonical`).
    *   Теги `hreflang` для поисковых роботов Google.
    *   Автоматическая разметка JSON-LD для сущности `Person` (на главной) и `BlogPosting` (в деталях статьи).

### Интерактивные фичи:
*   **Стилизованный терминал:** В подвале сайта (`footer`) расположен интерактивный эмулятор терминала со вкладками, соответствующими разделам сайта.
*   **BSOD Easter Egg:** Отдельная страница `/bsod`, полностью имитирующая синий экран смерти Windows с CRT-эффектом (сканирующие линии, мерцание) и таймером прогресса.
*   **Интерактивная форма обратной связи:** Форма связи на странице `/contact` отправляет данные на обработчик Formspree по AJAX. Внедрена защита от спама: невидимая ловушка (honeypot `_gotcha`) и временная задержка (блокировка отправки менее чем за 2 секунды с момента загрузки страницы).
*   **Защита почты (Обфускация):** Все упоминания email-адреса `almanex@zohomail.eu` защищены от парсеров-спамеров (данные хранятся в data-атрибутах и собираются JS на клиенте, в исходном коде отображается заглушка `[protected]`).

## 4. Ключевые решения и Костыли (Context & Workarounds)

> [!IMPORTANT]
> При работе с этим кодом необходимо учитывать следующие важные решения, чтобы избежать случайной поломки функционала при рефакторинге.

### 1. Конвертация ссылок на изображения в Markdown (`resolveSinglePath`)
В `src/components/IndexPage.astro` внедрена функция парсинга путей изображений из README проектов:
```javascript
function resolveSinglePath(src, project) {
  if (!src || src.startsWith('http://') || src.startsWith('https://') || src.startsWith('data:')) {
    if (src && src.startsWith('https://github.com/') && src.includes('/blob/')) {
      return src.replace('/blob/', '/raw/');
    }
    return src;
  }
  const baseRepo = project.github.replace(/\/$/, "");
  const baseRaw = baseRepo.replace("github.com", "raw.githubusercontent.com") + "/main/";
  const cleanSrc = src.replace(/^\.\//, "");
  return baseRaw + cleanSrc;
}
```
**Зачем это нужно:** README-файлы проектов содержат ссылки на медиа-ресурсы, предназначенные для отображения на GitHub. Данный парсер автоматически преобразует ссылки формата `/blob/` в `/raw/` и относительные пути в абсолютные ссылки на raw-репозиторий, благодаря чему картинки корректно отображаются на страницах портфолио.

### 2. Избежание мерцания темы в Head
Вызов скрипта инициализации темы оформлен в виде самовызывающейся функции (IIFE) непосредственно в начале `<head>` до отрисовки `<body>`:
```html
<script is:inline>
  (function(){
    const themeDefault = "light";
    try {
      document.documentElement.dataset.theme = localStorage.getItem("alex-theme") || themeDefault;
    } catch (e) {
      document.documentElement.dataset.theme = themeDefault;
    }
  })();
</script>
```
**Зачем это нужно:** Попытка перенести этот код во внешние JS-файлы или отложить его выполнение приведет к "миганию" белого фона при загрузке темной темы. Использование директивы `is:inline` критически важно.

## 5. Требования к окружению (Environment)

### Переменные окружения (`.env`):
На данный момент проект полностью статический и собирается без обязательных переменных окружения. Однако для переноса захардкоженных параметров рекомендуется внедрить:
*   `PUBLIC_FORMSPREE_ENDPOINT` — Эндпоинт формы обратной связи Formspree.

### Команды управления:
*   **Установка зависимостей:**
    ```bash
    npm install
    ```
*   **Запуск сервера разработки (в фоне):**
    ```bash
    astro dev --background
    ```
*   **Управление фоновым сервером:**
    *   Проверка статуса: `astro dev status`
    *   Просмотр логов: `astro dev logs`
    *   Остановка сервера: `astro dev stop`
*   **Сборка проекта (production):**
    ```bash
    npm run build
    ```
*   **Предпросмотр сборки:**
    ```bash
    npm run preview
    ```

## 6. Технический долг и Известные проблемы

1.  **Хардкод размеров файлов (Size Map):**
    В компонентах `src/components/IndexPage.astro` и `src/components/BlogDetailPage.astro` используется захардкоженная карта размеров файлов статей:
    ```javascript
    const sizeMap = {
      'vscode-agents': '5.2 KB',
      'wsl-containers': '12.4 KB'
    };
    ```
    Необходимо заменить динамическим подсчетом размера файлов `.md` при сборке проекта.
2.  **Захардкоженный эндпоинт Formspree:**
    В файле `src/components/ContactPage.astro` в атрибуте `action` формы отправки жестко прописан URL `https://formspree.io/f/xbdnrbrk`. Это затрудняет локальное тестирование формы с тестовыми эндпоинтами.
3.  **Монолитный файл стилей:**
    Файл `src/styles/global.css` имеет размер почти **78 KB**. Вся логика стилизации (переменные тем, анимации, скроллбары, разметка карточек, стили BSOD и терминала) находится в одном месте. Рекомендуется разделить стили на компонентные секции или импортировать модули CSS.
